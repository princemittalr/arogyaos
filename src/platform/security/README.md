# Platform Phase 6: Enterprise Security, Privacy, Compliance & Governance Platform

Global architectural Security & Compliance Configuration layer for ArogyaOS.

## Overview
This platform layer operates as the central metadata control plane defining permissions, data classification schemas, cryptographic mapping policies, and compliance boundaries across ArogyaOS's federated clinical domains.

## Architecture
**Constraint Warning**: This module exclusively administers **Metadata Configuration**. It does **not** process OAuth logins, issue JWTs, communicate directly with Keycloak, execute AES/RSA cryptographic operations, or operate physical SIEM dashboards. ArogyaOS dictates that the frontend manages the *hierarchies, rules, and privacy bounds*, fully abstracting the backend authorization execution engines.

## Folder Structure
- `types/` - Exacting types for security constraints (`RolePolicy`, `Consent`, `ComplianceFramework`).
- `core/` - Global security events, constants (e.g., PHI tags), and offline caching architecture.
- `utils/` - Zero-trust Zod schemas that validate all privacy and access permutations.
- `repositories/` - Boundary definitions for security metadata persistence.
- `services/` - Metadata orchestrators governing the creation of matrices and privacy rulings.
- `hooks/` - Abstracted React Query hooks linking configurations to IT control panels.
- `components/` - WCAG 2.2 AA compliant IT administration hubs allowing live compliance mapping.

## Extension Guidelines
To execute physical OIDC integrations, active packet scanning, or database encryption, establish isolated backend network services. Configure those runtimes to fetch and map against the strict metadata definitions (e.g., `EncryptionPolicy`, `PermissionMatrix`) generated by this platform UI.
